## ┬з4 Distribution & Deployment тАФ addition

```
www:
  Build command: pnpm build
  Output directory: .svelte-kit/cloudflare
  Adapter: @sveltejs/adapter-cloudflare
  Custom domain: alternatif.space   (apex)
```

- Fully prerendered: `prerender = true` on the root layout. Only the Clerk widget hydrates client-side (PRD ┬з7 тАФ "static, no JS beyond Clerk + minimal interaction").
- Environment variables: `PUBLIC_CLERK_PUBLISHABLE_KEY` only. **No Supabase dependency at launch** тАФ a live stat ("X partai aktif") is deferred because it would break full prerendering.
- **Native shell:** the landing is *not* added as a Capacitor tab тАФ app users are past the marketing layer.

## ┬з5 Auth & Cross-Subdomain Session тАФ addition

The landing is an **auth entry point**:

- Same Clerk instance and env var set as the other apps. The apex sits *inside* the `.alternatif.space` cookie scope, so session detection works without satellite-domain configuration.
- **Client-only Clerk** тАФ unlike partai/mufakat there is no `withClerkHandler` server hook (the app is fully prerendered; server hooks never run at request time). `<ClerkProvider>` initialises clerk-js after hydration, which reads the shared session cookie. Embedded components use `routing="hash"` because no catch-all route exists for path routing.
- Signed out тЖТ embedded `<SignUp />` in the hero with `forceRedirectUrl` тЖТ `https://partai.alternatif.space/onboarding` (partai owns post-signup onboarding; the landing never duplicates it). Secondary `/masuk` route hosts `<SignIn />` with `forceRedirectUrl` тЖТ partai root.
- **`/daftar` route (W-Q1: both, 2026-06-13)** тАФ dedicated `<SignUp />` page mirroring `/masuk`: canonical signup URL for share cards and press, target of mobile hero CTAs (replaces in-page anchor scroll), `noindex`, hash routing, same `forceRedirectUrl` тЖТ partai onboarding. The inline hero embed remains for desktop.
- Signed in тЖТ all auth CTAs swap to **"Lanjut ke partai тЖТ"** (partai's root decides between party page and onboarding; www cannot do a membership lookup тАФ no Supabase).

- Winner (**Pamflet visual ├Ч Arena copy** hybrid) is inlined directly in `src/routes/+page.svelte`. The prototype switcher, `src/lib/prototype/`, and all three variant components are deleted.
